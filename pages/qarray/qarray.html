A macro-based, ANSI C compliant dynamic array implementation. This array implementation
is designed to be:

  • Easy to use
  • Allow for both stack and heap operations
  • Typesafe
This library is quite simple to integrate into existing C projects. All the programemr
must do is include the header, define their types, and they can begin using the array.
This implementation also comes with all the expected operations that are implemented in
traditional arrays. A complete list can be found below.

Operations

  • Appending
  • Insertion
  • Popping
  • Removing
  • Searching

Important Notes

A tactic this implementation takes to operations is the settings macro defined for
each array. Each array type has a programmer-defined macro that carries settings, which
are passed as arguments to each macro. This is so it does not get too cumbersome for the
programmer.

In order to use this, you will need to include the following headers:
  • string.h
  • stdlib.h

Due to how macro arguments work, all array operations are actually wrappers around the
macro with the same name, but prefixed with double underscores. The programmer should not
make use of any macros prefixed with a double underscore.

Configuration

This array implementation defines the following compile-time options, which should be defined
before the header is included. If it is not defined, a default will be provided.
QARRAY_STACKFUL
If this macro is defined, the array operations will behave in a stackful manner. Nothing will
be done that involves heap memory.
QARRAY_RESIZE
If this macro is defined, the array will invoke this macro function to calculate the next size
of the array during a resize operation. If the array is stackful, this is not necessary.
An example (which is also the default) would be: 
            
    #define QARRAY_RESIZE(current_size) \
        current_size + 5
QARRAY_INITIAL_SIZE
If this macro is defined, it will be used as the initial size of a heap-allocated array. If
the array is stackful, this is not necessary. If it is not defined, it defaults to 5.
